# Databases Enhancement (DAD 220)

## Overview of the Artifact
This artifact originates from DAD 220: Introduction to SQL. The original project involved designing and creating a relational database using MySQL, building tables, defining relationships, and importing structured datasets. The purpose of the project was to demonstrate foundational SQL skills, including schema creation, data manipulation, and basic querying.

## Justification for Inclusion
I selected this artifact for my ePortfolio because it represents my growth in database design and management—an essential area of computer science. Databases are at the core of most software systems, and this enhancement allowed me to demonstrate my ability to work with structured data, enforce relationships, and build scalable, realistic database environments. The components of this artifact that best showcase my skills include table creation, schema refinement, data imports, and the use of SQL tools to validate and manage data.

## Summary of Enhancements
For this enhancement, I rebuilt the entire database environment from scratch using MySQL Workbench. I recreated all tables, re-established relationships, and imported expanded datasets to create a more realistic and complete database. One of the key improvements was adding an **Email** column to the Customers table, which reflects real-world business needs for communication and identity verification. I also verified data counts, ensured referential integrity, and tested the database to confirm that all tables and relationships were functioning correctly.

## Course Outcomes Addressed
This enhancement aligns with several Computer Science program outcomes. By redesigning the schema and rebuilding the database, I demonstrated my ability to design and evaluate computing solutions using appropriate database practices. This work also reflects my ability to use well-founded techniques and tools—such as MySQL Workbench and SQL import utilities—to implement solutions that deliver value and meet industry expectations. Additionally, adding the Email field and validating data imports reinforced the importance of designing systems with future security and data validation needs in mind.

## Process Reflection
Enhancing this artifact taught me a great deal about the importance of careful planning and attention to detail in database development. Rebuilding the environment required me to troubleshoot issues with data imports, resolve relationship conflicts, and ensure that the schema supported the expanded dataset. One challenge I faced was ensuring that all foreign key relationships were correctly defined after re-importing the data. I resolved this by reviewing the original schema, adjusting the order of table creation, and validating each relationship step-by-step. I also learned how small changes—such as adding a new column—can impact the overall structure and usability of a database. This enhancement strengthened my understanding of relational design, data integrity, and the practical considerations involved in maintaining and improving database systems.
